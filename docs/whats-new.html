01Four Inject Modes

v6.3 had two modes: Project XML and Blocks XML. v7.0 splits these into four distinct modes to give you precise control over what gets overwritten and what doesn't.

ModeInputWhat it doesDestructive?
[#] New Project <project> Wipes every sprite and builds a completely new project from the XML. Yes — all sprites
[+] Add to Project <project> Merges new sprites in. If a sprite with the same name already exists, its blocks and costumes are updated. Other sprites are untouched. Partial — named sprite only
~ Replace Blocks <xml> Clears the active sprite's block workspace and replaces it with the new XML. All other sprites untouched. Sprite blocks only
[+] Append Blocks <xml> Adds the new blocks to the active sprite's workspace without removing anything. Existing scripts are preserved. None
ℹ
The ! Example button is mode-aware — it loads a Project XML example when a project mode is active, and a Blocks XML example when a blocks mode is active.

02Add to Project (Merge Mode)

This is the most-requested v7.0 feature. Previously you had to wipe your entire project to inject new sprites. Now you can paste a <project> document and the injector will merge it into the running project.

Merge rules

SituationResult
Sprite in XML does not exist in project Sprite is created fresh with new costumes and blocks.
Sprite in XML has the same name as an existing sprite That sprite's blocks are replaced. Costumes are replaced if the XML includes any <costume> elements.
Existing sprite not mentioned in XML Completely untouched — its blocks, costumes, position, and variables are preserved.

This allows a powerful workflow: build your game project sprite by sprite, injecting one at a time without ever needing to rebuild the whole thing.

XML — Add a single new sprite to existing project
<!-- Mode: [+] Add to Project
     This adds "Bullet" without touching Player, Enemy, etc. -->
<project>
  <sprite name="Bullet" x="0" y="0">
    <costume type="circle" color="#ffff00" size="10"/>
    <blocks>
      <xml xmlns="https://developers.google.com/blockly/xml">
        <block type="event_whenflagclicked" x="40" y="40">
          <next>
            <block type="looks_hide"/>
          </next>
        </block>
      </xml>
    </blocks>
  </sprite>
</project>

03Append Blocks

The [+] Append Blocks mode adds new block stacks to the active sprite alongside existing ones — nothing is deleted.

Use this when you want to add extra scripts (like a new event handler or a helper stack) without losing the blocks you already have.

⚠
If you inject a stack with the same hat block (e.g. two event_whenflagclicked), Scratch will run both. This is intentional behaviour — the append mode does not deduplicate.
XML — Append a new script to existing sprite
<!-- Mode: [+] Append Blocks
     Adds this stack without removing any existing scripts -->
<xml xmlns="https://developers.google.com/blockly/xml">
  <block type="event_whenkeypressed" x="300" y="40">
    <field name="KEY_OPTION">r</field>
    <next>
      <block type="motion_gotoxy">
        <value name="X"><shadow type="math_number"><field name="NUM">0</field></shadow></value>
        <value name="Y"><shadow type="math_number"><field name="NUM">0</field></shadow></value>
      </block>
    </next>
  </block>
</xml>

04Export Full Project XML

The Export tab now has two buttons:

ButtonOutput
Grab Sprite XML The active sprite's Blockly blocks as a standard <xml> document. Same as v6.3.
Grab Project XML The entire project serialised as a <project> document — all sprites and their block scripts. Can be re-injected directly with [+] Add to Project or [#] New Project.

The Project XML exporter converts the live sb3 JSON block format back into readable Blockly XML, so you can save a project's XML state and re-inject it later — even on a different project.

ℹ
Exported Project XML includes <!-- costume: ... --> comments for each costume since the PNG data can't be round-tripped into shape attributes. To re-inject with correct costumes, add <costume type="..."> elements manually using the Costume Shapes guide.

05Copy Script Tab

A new </> Script tab in the panel lets you copy the injector's own userscript source code directly to your clipboard — no need to navigate back to Tampermonkey's dashboard.

When running as a Tampermonkey userscript, the tab shows instructions for how to access and copy the source from the Tampermonkey editor, since browser security prevents scripts from reading their own injected source. See the Copy Script guide for full details.

06Migrating from v6.3

v7.0 is a drop-in replacement. All v6.3 XML (both <xml> and <project> formats) continues to work without any changes.

v6.3 behaviourv7.0 equivalent
~ Blocks XML mode → Inject~ Replace Blocks mode
[#] Project XML mode → Build Project[#] New Project mode
Export → Grab XML (sprite only)Export → Grab Sprite XML

To install v7.0, open Tampermonkey Dashboard → click the script name → Edit → select all → paste the new source → save.
