Hosting on GitHub Pages

Deploy this wiki for free using GitHub Pages — no servers, no build tools, no cost.

Free ~10 min setup

01File Structure

The wiki is a self-contained static site. Here's the directory layout you'll upload:

File tree
scratch-xml-wiki/
├── index.html                  ← Landing / home page
├── assets/
│   ├── css/
│   │   └── main.css            ← Shared styles
│   └── js/
│       └── wiki.js             ← Shared scripts
└── docs/
    ├── getting-started.html
    ├── blocks-xml.html
    ├── opcodes.html
    ├── inputs-fields.html
    ├── project-xml.html
    ├── costumes.html
    ├── examples.html
    ├── troubleshooting.html
    └── hosting.html            ← This page

All paths are relative — the site works from any subfolder or custom domain without modification.

02Deploy via GitHub Web UI

The easiest method — no Git knowledge required.

1

Create a GitHub account

Go to github.com and sign up for a free account if you don't have one.

2

Create a new repository

Click the + icon (top right) → New repository. Name it scratch-xml-wiki. Set visibility to Public. Leave everything else as default. Click Create repository.

3

Upload files

Click Add file → Upload files. Drag the entire wiki folder structure into the upload area. GitHub preserves the folder structure automatically. Write a commit message (e.g. "initial wiki upload") and click Commit changes.

4

Enable GitHub Pages

In your repository, go to Settings → Pages. Under Build and deployment → Source, select Deploy from a branch. Set Branch to main, folder to / (root). Click Save.

5

Wait ~60 seconds, then visit your site

GitHub will show a banner with your URL once it's ready. Your wiki will be live at:

URL
https://YOUR-USERNAME.github.io/scratch-xml-wiki/
✓
Once deployed, any update you push to the main branch goes live within ~30–60 seconds automatically. No re-configuration needed.

03Deploy via Git CLI

Faster for ongoing updates once Git is installed on your machine.

Bash
# 1. Clone your newly created (empty) repository
git clone https://github.com/YOUR-USERNAME/scratch-xml-wiki.git
cd scratch-xml-wiki

# 2. Copy all wiki files into the folder
#    (index.html, assets/, docs/) then commit and push
git add .
git commit -m "initial wiki upload"
git push origin main

# For subsequent updates:
git add .
git commit -m "update opcodes table"
git push

04Custom Domain (Optional)

If you own a domain (e.g. mysite.com), you can point it at your GitHub Pages site.

1

Add a CNAME file

Create a file named exactly CNAME (no extension) in the root of the repository. Its only content should be your domain:

CNAME
wiki.mysite.com
2

Update your DNS

In your domain registrar's DNS settings, add a CNAME record pointing wiki.mysite.com → YOUR-USERNAME.github.io.

3

Enable in GitHub Pages settings

In Settings → Pages → Custom domain, enter your domain. GitHub will issue a free TLS certificate automatically within a few minutes.

05Tips

ℹ
No build step required. This wiki is plain HTML, CSS, and JS — GitHub Pages serves it directly. You don't need Node, npm, Jekyll, or any other tooling.
ℹ
Private repos require GitHub Pro for Pages. If you want to keep the source private but host publicly, you'll need a paid plan. Public repos get Pages for free on any plan.
