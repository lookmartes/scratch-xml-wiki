Inputs & Fields

A focused guide on writing every type of input and field correctly — numbers, strings, booleans, dropdowns, and colours.

Core concept

01Number Input

The most common input pattern. Use a math_number shadow inside a <value>:

XML
<value name="STEPS">
  <shadow type="math_number">
    <field name="NUM">10</field>
  </shadow>
</value>

To use a negative number, simply write a negative value in <field name="NUM"> — e.g. -5.

Other number shadow variants: math_positive_number, math_whole_number, math_integer, math_angle. All use the same NUM field name.

02Text / String Input

XML
<value name="MESSAGE">
  <shadow type="text">
    <field name="TEXT">Hello, World!</field>
  </shadow>
</value>
⚠
Avoid non-ASCII characters (smart quotes, emoji, accented letters) in field values. These can cause FixedAsciiString errors. Use only plain ASCII text or click "Strip Non-ASCII" in the panel.

03Boolean Input

Boolean values (hexagonal blocks) are placed as real <block> elements inside a <value>. Never as a <shadow>.

XML — Key pressed boolean
<value name="CONDITION">
  <block type="sensing_keypressed">
    <value name="KEY_OPTION">
      <shadow type="sensing_keyoptions">
        <field name="KEY_OPTION">space</field>
      </shadow>
    </value>
  </block>
</value>
XML — Compound boolean (A and B)
<value name="CONDITION">
  <block type="operator_and">
    <value name="OPERAND1">
      <block type="sensing_keypressed">
        <value name="KEY_OPTION">
          <shadow type="sensing_keyoptions">
            <field name="KEY_OPTION">up arrow</field>
          </shadow>
        </value>
      </block>
    </value>
    <value name="OPERAND2">
      <block type="sensing_mousedown"/>
    </value>
  </block>
</value>

04Dropdown Fields

Some blocks expose a dropdown as a <field> directly on the block element. Here are the valid values for the most common dropdowns:

KEY_OPTION — keyboard keys

spaceleft arrowright arrowup arrowdown arrow enterabcde fghijkl mnopqrs tuvwxyz 0123456789

STOP_OPTION — control_stop

allthis scriptother scripts in sprite

05Reporter Block as Input

To use a reporter block (like sensing_timer or motion_xposition) as an input value, place it as a <block> alongside a <shadow>:

XML — Reporter + shadow
<value name="NUM">
  <!-- The real block (reporter) -->
  <block type="sensing_timer"/>
  <!-- Shadow is still provided as fallback -->
  <shadow type="math_number">
    <field name="NUM">0</field>
  </shadow>
</value>
ℹ
When both a <block> and <shadow> are present in a <value>, the real block is used and the shadow acts as the hidden fallback — matching how Scratch stores it internally.
