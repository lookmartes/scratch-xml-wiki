Opcode Reference

Every common Scratch block opcode with its type string, input names, and field values. Use these as the type attribute on <block> elements.

Reference
ℹ
Each opcode listed here is the exact string to use as type="…" on a <block> element. Value inputs go in <value name="…">. Fields go in <field name="…">.

Variables

⚠
Variable and list blocks require matching variable definitions in the project's targets JSON. When using Blocks XML mode against an existing project, existing variables work. In Project XML mode, variables must be declared in the sprite's target data — this is an advanced use case not fully supported by the current XML format.
OpcodeBlockNotes
data_setvariabletoSet [ ] to ( )field VARIABLE, value VALUE
data_changevariablebyChange [ ] by ( )field VARIABLE, value VALUE
