Export Modes

v7.0 adds a second export button — grab just the active sprite's blocks, or export your entire project as re-injectable XML.

v7.0Reference

Overview

The [cp] Export tab has two grab buttons and one copy button:

ButtonOutput formatScope
Grab Sprite XML <xml xmlns="..."> Active sprite's blocks only
Grab Project XML <project><sprite>...</sprite></project> All sprites in the entire project
[cp] Copy to Clipboard — Copies whatever is in the output box

01Grab Sprite XML

Reads the current sprite's Blockly workspace and serialises it to a <xml> Blockly document. This is identical to the v6.3 export behaviour.

Use cases

The most common use is learning: build blocks in Scratch's visual editor, then grab the XML to understand what the structure looks like. You can then hand-write similar XML and inject it back.

You can also use it as a backup/restore — grab a sprite's XML before running Replace Blocks, then re-inject it if something goes wrong.

Output example

Sprite XML output (example)
<xml xmlns="https://developers.google.com/blockly/xml">
  <block type="event_whenflagclicked" x="40" y="40">
    <next>
      <block type="motion_movesteps">
        <value name="STEPS">
          <shadow type="math_number">
            <field name="NUM">10</field>
          </shadow>
        </value>
      </block>
    </next>
  </block>
</xml>

02Grab Project XML

Reads the entire live project from the Scratch VM, converts every sprite's sb3 block JSON back to readable Blockly XML, and wraps everything in a <project> document.

Output structure

Project XML output (structure)
<project>
  <sprite name="Player" x="0" y="-100" visible="true">
    <!-- costume: player (png) -->
    <blocks>
      <xml xmlns="https://developers.google.com/blockly/xml">
        <block type="event_whenflagclicked" x="40" y="40">
          <!-- ... blocks ... -->
        </block>
      </xml>
    </blocks>
  </sprite>
  <sprite name="Enemy" x="80" y="60" visible="true">
    <!-- costume: enemy (png) -->
    <blocks>
      <!-- ... -->
    </blocks>
  </sprite>
</project>

Costume comments

Because PNG costume data cannot be losslessly round-tripped back into shape attributes, each costume is represented as a comment: <!-- costume: name (format) -->. To make the exported XML re-injectable with correct costumes, add <costume type="..."> elements manually above the <blocks> element.

Re-injecting exported XML

The exported <project> XML can be pasted directly back into the Inject tab in either [#] New Project or [+] Add to Project mode. The blocks will be restored exactly.

ℹ
This is the recommended way to save and restore a project's script state. Grab → Copy → save to a text file → re-inject any time.

03Full Copy Workflow

1

Open the Export tab

Click [cp] Export in the panel's tab bar.

2

Choose what to export

Click Grab Sprite XML for the active sprite only, or Grab Project XML for the whole project.

3

Copy to clipboard

Click [cp] Copy to Clipboard. The button flashes green when the copy succeeds.

4

Paste anywhere

Paste into a text editor, another project's injector, or the wiki's example box.
